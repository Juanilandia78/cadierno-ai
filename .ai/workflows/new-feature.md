# Workflow - New Feature

## Metadata

**Prioridad:** Alta

**Especialista Líder:** Architect

**Especialistas participantes**

- Backend
- Database
- DevOps
- QA
- Reviewer
- Documentation

---

## Objetivo

Implementar una nueva funcionalidad respetando la arquitectura del proyecto y los principios de Cadierno AI.

---

## Cuándo utilizar este Workflow

Utilizar cuando el usuario solicite:

- Nueva funcionalidad.
- Nuevo módulo.
- CRUD.
- Integración con APIs.
- Integración con servicios externos.
- Automatizaciones.
- Nuevos procesos de negocio.

---

## Flujo

> Para trabajo de riesgo o varios escenarios, seguir el playbook de
> [Ejecución por Microtareas](../../playbooks/ai-agents/microtask-execution.md).
> Código escrito no es cierre: cada paso requiere evidencia y auditoría final.

### 1. Architect

- Comprender el problema.
- Analizar impacto.
- Diseñar estrategia.
- Detectar Specialists involucrados.

---

### 2. Backend

- Implementar la funcionalidad.
- Crear Tests cuando corresponda.
- Mantener consistencia.

---

### 3. Database

Si aplica:

- Migraciones.
- Índices.
- Optimización.

---

### 4. DevOps

Si aplica:

- Variables.
- Docker.
- Infraestructura.
- Deploy.

---

### 5. QA

Validar:

- Casos felices.
- Casos negativos.
- Permisos.
- Validaciones.
- Regresiones.

---

### 6. Reviewer

Revisar:

- Calidad.
- Arquitectura.
- Simplicidad.
- Seguridad.

---

### 7. Documentation

Actualizar:

- README
- CHANGELOG
- APIs
- Knowledge

---

## Criterios de aceptación

✓ Funciona

✓ Está testeado

✓ Fue revisado

✓ Está documentado

✓ Respeta la arquitectura
