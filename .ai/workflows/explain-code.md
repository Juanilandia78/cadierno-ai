# Workflow - Explain Code

## Objetivo

Explicar código existente de forma clara, accionable y útil para mantenimiento.

Este workflow se usa para:

- onboarding técnico;
- análisis de módulos complejos;
- preparación antes de bugfix/refactor;
- transferencia de conocimiento.

---

## Paso 1 - Delimitar el análisis

Definir:

- archivo/módulo objetivo;
- nivel de detalle esperado;
- foco (negocio, técnico, performance, seguridad).

---

## Paso 2 - Entender contexto del proyecto

Consultar:

- knowledge/project.md
- knowledge/architecture.md
- knowledge/decisions.md

---

## Paso 3 - Explicar estructura

Describir:

- propósito del archivo o módulo;
- responsabilidades principales;
- dependencias relevantes;
- entradas/salidas.

---

## Paso 4 - Explicar flujo de ejecución

Detallar:

- camino feliz;
- validaciones;
- casos de error;
- side effects.

---

## Paso 5 - Identificar riesgos

Marcar:

- acoplamientos peligrosos;
- código difícil de mantener;
- posibles bugs;
- oportunidades de mejora.

---

## Paso 6 - Traducir a acciones

Cerrar con:

- recomendaciones concretas;
- quick wins;
- próximos pasos.

---

## Formato recomendado de salida

1. Resumen funcional.
2. Flujo paso a paso.
3. Riesgos y deuda técnica.
4. Mejoras sugeridas.

---

## Resultado esperado

El desarrollador debe poder:

- comprender rápidamente el módulo;
- estimar impacto de cambios;
- planificar una intervención segura.
