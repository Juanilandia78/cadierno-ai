# Backend Checklist

## Antes de comenzar

- Comprender el requerimiento funcional.
- Revisar la arquitectura existente.
- Buscar implementaciones similares.
- Verificar si ya existe lógica reutilizable.

---

## Durante el desarrollo

- Utilizar validaciones mediante Form Requests.
- Mantener Controllers livianos.
- Extraer lógica de negocio a Services cuando corresponda.
- Respetar SOLID y las convenciones del proyecto.
- Evitar duplicación de código.
- Manejar correctamente excepciones y errores.
- Registrar logs cuando aporten valor.

---

## Antes de finalizar

- Código legible.
- Nombres descriptivos.
- Validaciones completas.
- Autorización implementada.
- Tests agregados o actualizados.
- Documentación actualizada si corresponde.
- Sin código comentado ni muerto.

---
