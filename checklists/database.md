# Database Checklist

## Diseño

- Tablas correctamente normalizadas.
- Tipos de datos adecuados.
- Claves foráneas definidas.
- Restricciones necesarias.

---

## Rendimiento

- Índices revisados.
- Consultas optimizadas.
- EXPLAIN analizado cuando sea necesario.
- Evitar N+1.

---

## Seguridad

- Validar integridad de datos.
- Evitar consultas inseguras.
- Utilizar transacciones cuando corresponda.

---

## Antes de finalizar

- Migraciones correctas.
- Rollback probado.
- Relaciones verificadas.

---

## Filosofía Cadierno AI

Una base de datos bien diseñada simplifica todo el sistema.
