# Roadmap - Cadierno AI

> Pensá. Diseñá. Construí. Revisá. Evolucioná.

Este documento describe la evolución planificada de Cadierno AI.

---

# Visión

Cadierno AI busca convertirse en un framework de ingeniería asistida por Inteligencia Artificial.

No pretende reemplazar al desarrollador.

Pretende convertir a cualquier desarrollador en un mejor ingeniero de software.

---

# V1.0 - Core

## Objetivo

Crear la base del framework.

### Incluye

- Filosofía (CADIERNO.md)
- Specialists
- Workflows
- Styles
- Bootstrap
- Knowledge Templates
- Checklists
- Playbooks
- README
- Instalación

Estado: 🚧 En desarrollo

---

# V1.1 - Knowledge & Memory

## Objetivo

Que Cadierno AI recuerde el contexto de cada proyecto.

### Incluye

- Generación automática de `knowledge/project.md`
- Generación de `knowledge/patterns.md`
- Generación de `knowledge/decisions.md`
- Historial de sesiones
- Memoria local basada en Markdown

Estado: ⏳ Planificado

---

# V1.2 - Intelligent Memory

## Objetivo

Separar Cadierno AI del proveedor de memoria.

### Incluye

- Memory Providers
- Integración con Mem0
- Sistema de resúmenes automáticos
- Aprendizaje continuo

Estado: ⏳ Planificado

---

# V2.0 - Cadierno Console

## Objetivo

Controlar el framework mediante comandos.

Ejemplos:

/cad bootstrap

/cad audit

/cad feature

/cad review

/cad bugfix

/cad refactor

/cad memory

Estado: 💡 Idea

---

# V3.0 - AI Engineering Platform

## Objetivo

Convertir Cadierno AI en una plataforma completa.

### Posibles características

- Subagentes automáticos
- MCPs
- Marketplace de Playbooks
- Dashboard
- CLI
- Extensión para VS Code

Estado: 🔮 Futuro

---

# Nuestra prioridad

Antes de agregar nuevas funcionalidades siempre debemos preguntarnos:

- ¿Hace pensar mejor a la IA?
- ¿Hace trabajar mejor a la IA?
- ¿Hace recordar mejor a la IA?

Si la respuesta es "no", probablemente no pertenezca a Cadierno AI.
