# Playbook - Redis

## Objetivo

Utilizar Redis para mejorar el rendimiento del sistema.

---

## Casos de uso

- Cache
- Queues
- Sessions
- Locks
- Rate Limiting

---

## Buenas prácticas

- Evitar almacenar información permanente.
- Configurar expiraciones cuando corresponda.
- No utilizar Redis como base de datos principal.

---

## Checklist

✓ Conexión configurada

✓ Variables de entorno

✓ Cache funcionando

✓ Queue funcionando

---

## Filosofía

Redis acelera la aplicación.

No reemplaza la base de datos.
