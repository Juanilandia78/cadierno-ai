# Playbook - Release

## Objetivo

Publicar nuevas versiones de forma controlada.

---

## Antes de liberar

- Tests
- QA
- Backup
- Changelog
- Tag

---

## Checklist

✓ QA aprobado

✓ Changelog actualizado

✓ Tag creado

✓ Deploy validado

---

## Filosofía

Cada Release debe poder revertirse rápidamente.
