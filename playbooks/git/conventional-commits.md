# Playbook - Conventional Commits

## Objetivo

Mantener un historial de Git claro y consistente.

---

## Formato

tipo(scope): descripción

---

## Tipos

feat

fix

docs

style

refactor

test

chore

perf

ci

build

---

## Ejemplos

feat(auth): agregar login con Google

fix(api): corregir validación de usuarios

docs(readme): actualizar instalación

---

## Filosofía

Un buen commit explica el cambio sin necesidad de leer el código.
